Get Involved

MAPI is being built in public. There's a lot of work ahead to prove the concept, improve the spec, and enable real-world adoption.

Building in Public

MAPI is a technology preview. We believe the core ideas are sound, but we need the community's help to validate them, expand the test suite, build tooling, and get Skills into the hands of agent developers.

We're not trying to own this. We want to embrace existing standards like Agents.md and AgentSkills.io, and ultimately contribute MAPI to a standards body once it's proven in the real world.

Where We Need Help

These are opportunities we've identified but don't have bandwidth to pursue ourselves. If any of these interest you, we'd love to collaborate.

Reference Skills for Popular APIs High Impact

People learn by example. Having battle-tested Skills for popular APIs makes adoption tangible.

We've built Skills for Anthropic, GitHub, and GCP Billing. The ecosystem needs Skills for Stripe, Twilio, Slack, OpenAI, Notion, Linear, Supabase, and more. Each Skill should include E2E tests that prove it actually works against the live API.

VS Code Extension

Developers live in their IDE. Good DX signals a mature ecosystem.

An extension that provides syntax highlighting for .mapi.md and Skill.md files, validation against the spec, and commands like "Convert OpenAPI to Skill" would make authoring Skills much easier.

Skill Loader Library

Skills are documentation until agents can consume them programmatically.

A library that fetches a Skill from a URL, parses its capabilities, and returns structured data that agent frameworks can use. This is the bridge between "Skill as documentation" and "Skill as executable spec."

Registry & Aggregator Support

Discovery drives adoption. If people can't find Skills, they won't use them.

Platforms like Smithery.ai, Glama, and PulseMCP index MCP servers. We'd love to see Skills indexed alongside them. We can define a skill-manifest.json schema to make indexing easy, but we need registry operators to adopt it.

Well-Known Self-Publishing

API providers shouldn't depend on registries to publish their Skills.

A /.well-known/skill/Skill.md convention would let any API provider self-host their Skill at a predictable URL. Agents and crawlers could discover Skills automatically, just like robots.txt or security.txt.

Agent Framework Integrations

If LangChain, LlamaIndex, or Claude Code natively understand Skills, adoption follows.

Once a skill-loader library exists, we need adapters for popular agent frameworks. Show that Skills can be loaded and converted to tool definitions with minimal friction.

Standards Alignment

Our Approach to Standardization

We're not racing to submit MAPI to a standards body. Instead, we believe the right path is to embrace existing community standards like Agents.md and AgentSkills.io, prove the concepts work through real-world adoption, and then consider formal standardization once the spec has been battle-tested. If you're working on adjacent standards, we'd love to talk.
